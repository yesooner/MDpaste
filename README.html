MDPASTE Portable

基于上游 PasteMD 的 Windows 便携版封装，把 Markdown 转换成 Word/WPS/Office 格式

便携版 内置 Pandoc AGPL-3.0

是什么

MDPASTE Portable 是基于上游 PasteMD 制作的 Windows 便携版封装，用于把复制出来的 Markdown 内容转换成更适合粘贴到 Word、WPS、Office 等软件中的格式。

常见使用场景：从 AI 对话中复制回答内容，例如标题、列表、代码块、表格、公式等 Markdown 文本，然后通过 MDPASTE 转换并粘贴到文档里，减少手动排版。

本便携包已经内置 Pandoc，使用者不需要单独安装 Python、Pandoc 或其他命令行工具。

首次使用

  1. 在 GitHub Releases 页面下载 MDPASTE-portable-v0.1.0.0.zip。
  2. 解压到任意文件夹。
  3. 双击 MDPASTE.cmd 启动。
  4. 复制 Markdown 内容。
  5. 按默认快捷键 Ctrl+Alt+B 进行转换/粘贴。

不要直接双击 MdPaste.exe。请使用 MDPASTE.cmd，因为它会先准备便携数据目录，并按当前电脑路径自动写入配置。

文件说明

文件说明
MDPASTE.cmd普通用户启动入口，双击这个文件
MdPaste-portable.cmd实际便携启动脚本，负责设置路径和检查依赖

路径和移植

可以直接移植到新电脑。每次从 MDPASTE.cmd 启动时，程序都会按当前文件夹重写配置：

  • APPDATA → portable-data\Roaming
  • LOCALAPPDATA → portable-data\Local
  • pandoc_path → 当前文件夹下的 _internal\pandoc\pandoc.exe
  • save_dir → 当前文件夹下的 cache

所以把整个文件夹复制到另一台 Windows 电脑后，仍然双击 MDPASTE.cmd 即可，不需要手动修改路径。

Pandoc 使用

PasteMD 的文档/富文本转换依赖 Pandoc。本便携包已经把 Pandoc 放在：

_internal\pandoc\pandoc.exe

不要删除 _internal 目录。如果 Pandoc 缺失，启动脚本会停止并提示重新下载完整 ZIP。

开机自启

运行 switch-startup.cmd，按提示启用或关闭开机自启。

如果移动了文件夹，请重新运行一次 switch-startup.cmd，让 Windows 计划任务更新到新的路径。

本地数据

这些文件会在使用者电脑上自动生成：

  • 配置文件：portable-data\Roaming\PasteMD\config.json
  • 日志文件：portable-data\Roaming\PasteMD\pastemd.log
  • 缓存目录：cache

这些是本机运行数据，不提交到 Git，也不会打入干净的发布 ZIP。

版本维护

如果发行版本发生变化，需要同步修改以下位置：

  • README.md 和 README.txt 中的下载文件名
  • RELEASE_NOTES.md 中的版本号和附件名
  • SOURCE.md 中对应的上游 tag、commit 和源码链接
  • build-release.ps1 中的默认 $Version
  • Git tag，例如 v0.1.0.0
  • GitHub Release 标题、说明和附件 ZIP 名称

版本号不一致会导致用户下载说明、Release 附件和源码说明互不对不上，所以每次发布新版本都应统一检查。

MDPASTE Portable

A Windows portable build of the upstream PasteMD — convert Markdown to Word/WPS/Office formats with one shortcut

Portable Built-in Pandoc AGPL-3.0

What It Is

MDPASTE Portable is a self-contained Windows build of the upstream PasteMD project. It converts Markdown clipboard content into formats that paste cleanly into Word, WPS, and other Office applications.

Typical use case: copy a response from an AI chat — headings, lists, code blocks, tables, equations and all — then hit one shortcut to transform and paste it into your document without manual reformatting.

Pandoc is bundled in. No separate installation of Python, Pandoc, or any command-line tools is required.

Getting Started

  1. Download MDPASTE-portable-v0.1.0.0.zip from the GitHub Releases page.
  2. Extract to any folder.
  3. Double-click MDPASTE.cmd to launch.
  4. Copy some Markdown content.
  5. Press the default shortcut Ctrl+Alt+B to convert and paste.

Do not double-click MdPaste.exe directly. Always use MDPASTE.cmd, which sets up the portable data directory and rewrites all paths relative to the current folder location.

Portability & Path Handling

Move the folder to a new machine and it just works. Every time MDPASTE.cmd runs, it rewrites all paths to be relative to the current folder:

  • APPDATA → portable-data\Roaming
  • LOCALAPPDATA → portable-data\Local
  • pandoc_path → _internal\pandoc\pandoc.exe in the current folder
  • save_dir → cache in the current folder

Copy the entire folder to any other Windows PC, double-click MDPASTE.cmd, and you're ready to go — no manual path editing needed.

Using Pandoc

PasteMD relies on Pandoc for document and rich-text conversion. In this portable build, Pandoc lives at:

_internal\pandoc\pandoc.exe

Do not delete the _internal folder. If Pandoc is missing, the launch script will abort and prompt you to re-download the complete ZIP.

Auto-Start

Run switch-startup.cmd and follow the prompts to enable or disable auto-start on Windows boot.

If you move the folder to a different location, run switch-startup.cmd again so the Windows scheduled task updates to the new path.

Local Data

These files are generated automatically on the user's machine:

  • Config: portable-data\Roaming\PasteMD\config.json
  • Logs: portable-data\Roaming\PasteMD\pastemd.log
  • Cache directory: cache

These are local runtime artifacts — not committed to Git and excluded from clean release ZIPs.

Version Maintenance

When the release version changes, the following locations must be kept in sync:

  • Download filename in README.md and README.txt
  • Version number and attachment name in RELEASE_NOTES.md
  • Upstream tag, commit, and source link in SOURCE.md
  • Default $Version in build-release.ps1
  • Git tag, e.g. v0.1.0.0
  • GitHub Release title, description, and attachment ZIP name

Inconsistent version numbers cause a mismatch between what users download, what the release notes say, and what the source documentation references. Always audit all of these together when cutting a new release.

Repository & Release Process

The Git repository holds only launch scripts, build scripts, documentation, licenses, and source references. The full runnable package is distributed as a GitHub Release attachment.

To build a release ZIP locally:

powershell -NoProfile -ExecutionPolicy Bypass -File .\build-release.ps1

Output:

dist\MDPASTE-portable-v0.1.0.0.zip
